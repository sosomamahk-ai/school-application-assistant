# 🎯 最简单方法：看图操作

## 📸 步骤截图说明

### 第1步：打开PowerShell
```
按键盘：Win键 + X
然后点击：Windows PowerShell
```

### 第2步：输入命令进入项目
在PowerShell窗口中，你会看到类似这样的提示符：
```
PS C:\Users\你的用户名>
```

输入以下命令（复制粘贴即可）：
```
cd C:\school-application-assistant
```

然后按**回车键**，提示符会变成：
```
PS C:\school-application-assistant>
```

### 第3步：运行同步脚本
输入以下命令：
```
.\scripts\sync-schools.ps1
```

按**回车键**。

### 第4步：输入你的信息
脚本会问你两个问题：

**问题1：请输入管理员邮箱**
```
请输入管理员邮箱: 
```
在这里输入你的邮箱，比如：`admin@example.com`
然后按**回车键**

**问题2：请输入密码**
```
请输入密码: 
```
在这里输入你的密码（输入时不会显示任何字符，这是正常的，为了安全）
然后按**回车键**

### 第5步：等待完成
脚本会自动工作，你会看到类似这样的输出：
```
========================================
WordPress学校数据同步
========================================

步骤1: 正在登录...
✅ 登录成功！用户: admin@example.com

步骤2: 正在同步WordPress学校数据...
这可能需要几分钟时间，请耐心等待...

========================================
✅ 同步完成！
========================================

同步结果:
  ✓ 成功同步: 150 所学校
  ⚠ 错误数量: 0
  ⏱ 耗时: 5234ms

下一步: 访问 /schools 页面验证所有学校都显示了 name_short
```

## ❓ 如果出错了怎么办？

### 错误1：提示"无法运行脚本"
**解决方法**：
1. 在PowerShell中输入：
   ```
   Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
   ```
2. 按回车
3. 输入 `Y` 然后按回车
4. 再次运行脚本

### 错误2：提示"不是管理员"
**解决方法**：
- 联系开发人员，让他们把你的账户设置为管理员
- 或者告诉他们你的邮箱，让他们帮你设置

### 错误3：提示"登录失败"
**解决方法**：
- 检查邮箱和密码是否正确
- 确认你的账户存在
- 如果忘记密码，联系开发人员重置

### 错误4：找不到文件
**解决方法**：
1. 确认你在正确的目录（应该看到 `PS C:\school-application-assistant>`）
2. 如果不在，先运行：`cd C:\school-application-assistant`
3. 确认文件存在：输入 `dir scripts\sync-schools.ps1` 应该能看到文件
